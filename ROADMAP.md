# Roadmap

The roadmap is intentionally outcome-based. Items become implementation issues only after their user value, memory impact, and acceptance criteria are clear.

## Now: trustworthy first release

- Reproducible Windows build and release pipeline
- Versioned release assets, SHA-256 manifest, and provenance attestation
- Clean installation and notification-hook documentation
- Real-world reports from mixed-DPI and multi-monitor systems
- Keyboard and high-contrast audit of the settings and calibration flows

## Next: easier ownership

- In-app installation and safe notification-hook chaining
- Reduced-motion and static-notification modes
- Importable/exportable color and animation presets
- Localization-ready UI strings
- Structured diagnostic export with private content excluded
- Signed Windows distribution when a sustainable signing path is available

## Later: broader integrations

- Stable adapters for additional agent-completion event sources
- Community-contributed patterns with bounded rendering costs
- Optional installer/update channel
- Expanded accessibility automation and clean-VM test coverage

## How to propose work

Open an [Idea discussion](https://github.com/ajhcs/codex-edge-glow/discussions/categories/ideas) for large experience changes. Use the feature issue form for a bounded request with a concrete user outcome. A roadmap mention is not a promise of scope or timing.
